# Web App 화면 구성도
## Life Asset Planner – Web Screen & UI Structure

본 문서는 Life Asset Planner 웹 애플리케이션의  
**전체 화면 구조, 핵심 화면 구성, UI/UX 원칙**을 하나의 문서로 정의한다.

서비스는 **Daily Page(하루 단위 기록)**를 중심으로 설계된다.

---

## 1. 전체 화면 구조 개요


[ Global Layout ]
 ├─ Header (상단 고정)
 ├─ Sidebar (좌측, Desktop)
 └─ Main Content (중앙)
````

* 중심 개념: **Today / Daily Dashboard**
* 사용자는 항상 “오늘”로 빠르게 복귀 가능해야 한다
* 화면 이동보다 **한 화면 내 맥락 유지**를 우선한다

---

## 2. 공통 레이아웃 구성

### 2.1 Header (상단 영역)

**구성 요소**

* 서비스 로고 / 이름
* 오늘 날짜
* 빠른 액션

  * 오늘으로 이동
  * 다크 / 라이트 모드 토글
* 사용자 메뉴

  * 프로필
  * 로그아웃

**설계 원칙**

* 높이 최소화
* 항상 고정(sticky)
* 시선 방해 요소 제거

---

### 2.2 Sidebar (좌측 메뉴, Desktop 기준)

```text
[ Sidebar ]
- Today
- History
- Reports
- Settings
```

**원칙**

* 메뉴 수 최소화
* 아이콘 + 텍스트 병행
* Mobile 환경에서는 Drawer 형태로 전환

---

## 3. 핵심 화면 – Daily Dashboard

Daily Dashboard는 **서비스의 시작이자 끝**이다.
하루의 모든 기록은 이 화면에서 입력·확인·회고된다.

```text
[ Daily Dashboard ]
 ├─ 오늘의 요약
 ├─ To-Do
 ├─ 수입 / 지출
 ├─ 투자 기록
 └─ 메모 / 회고
```

---

### 3.1 오늘의 요약 영역

**표시 정보**

* 오늘 날짜
* 총 수입
* 총 지출
* 간단 자산 흐름 요약

**UI 원칙**

* 숫자는 크고 명확하게
* 색상은 의미 전달용으로만 사용
* 감정적 자극 최소화

---

### 3.2 To-Do 리스트 영역

**구성**

* 할 일 목록
* 상태 표시

  * 대기
  * 진행
  * 완료
* 중요도 표시
* 할 일별 메모

**UX 원칙**

* 완료된 항목은 흐리게 표시
* 삭제보다 “완료 상태 유지”
* 빠른 추가 / 빠른 체크

---

### 3.3 수입 · 지출 관리 영역 (KRW / USD)

**수입**

* 금액
* 통화 선택
* 수입 유형 (급여, 부수입, 투자수익 등)
* 메모

**지출**

* 금액
* 통화 선택
* 카테고리 (식비, 교통, 구독 등)
* 고정 / 변동 지출 구분
* 메모

**UI 원칙**

* 금액이 가장 먼저 보이도록 배치
* 통화는 보조 정보
* 숫자보다 “소비/수입의 이유” 기록 강조

---

### 3.4 환율 정보 (보조 요소)

* 기준 통화: KRW
* USD 환율 표시
* 수입 / 지출 입력 시 자동 환산

**설계 원칙**

* 기록 시점 환율 고정
* 과거 데이터는 변하지 않음
* 사용자가 직접 수정하지 않음

---

### 3.5 투자 기록 영역 (국내 / 미국 주식)

**표시 항목**

* 종목명 / 티커
* 매수 / 매도 구분
* 수량 / 체결 가격
* 통화 (KRW / USD)

**메모**

* 매수 이유
* 매도 판단 근거
* 당시 상황 기록

**UI 원칙**

* 수익률 강조 ❌
* 판단 메모 강조 ⭕
* 숫자보다 맥락 중심

---

### 3.6 메모 / 회고 영역

**구성**

* 자유 메모
* 하루 회고

  * 잘한 판단
  * 아쉬운 선택
  * 개선 포인트

**설계 원칙**

* 강제 입력 없음
* 부담 없는 분량
* 기록의 연속성 중시

---

## 4. 보조 화면 구성

### 4.1 History (과거 기록 조회)

```text
[ History ]
- 날짜 선택
- Daily Page 목록
```

* 날짜별 기록 조회
* 기본은 읽기 전용
* 수정은 최소화

---

### 4.2 Reports (주간 / 월간)

**제공 정보**

* 총 수입 / 총 지출
* 통화별 자산 흐름
* 지출 카테고리 요약
* 투자 기록 요약

**원칙**

* 분석 도구 ❌
* 회고 도구 ⭕

---

### 4.3 Settings

* 기본 통화 설정
* 다크 / 라이트 모드
* 계정 관리
* 데이터 내보내기

---

## 5. 화면 설계 핵심 원칙

* 모든 데이터는 Daily Page에 귀속
* 한 화면 = 하나의 맥락
* 입력은 빠르게, 조회는 차분하게
* 숫자보다 기록의 흐름을 보여준다
